# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с кодом в этом репозитории.

## Обзор проекта

Витрина цветочного магазина + админ-панель на Laravel 10 ("Vdohnovenie" / "diplom" — дипломный студенческий проект). Рендеринг на сервере через Blade-шаблоны, без SPA-фреймворка. В каталоге `vdohnovenie-flowers/` лежит дублирующая/параллельная копия приложения (свой composer.json, artisan, app/ и т.д.) — перед внесением изменений проверяй, в какой директории на самом деле нужно работать, так как копии могут расходиться независимо друг от друга.

## Команды

Выполняются из корня репозитория (или из `vdohnovenie-flowers/`, если работа ведётся в этой копии):

- Установка PHP-зависимостей: `composer install`
- Установка JS-зависимостей: `npm install`
- Сборка фронтенда в режиме разработки (Vite, watch): `npm run dev`
- Продакшн-сборка: `npm run build`
- Запуск приложения: проект работает под OSPanel (`.osp/project.ini`), а не через `artisan serve` — считай, что веб-сервер уже настроен на `public/`.
- Запуск тестов: `vendor/bin/phpunit` (или `php artisan test`)
- Запуск одного теста: `vendor/bin/phpunit --filter TestName tests/Feature/SomeTest.php`
- Миграции БД: `php artisan migrate`
- Линт/форматирование PHP: `vendor/bin/pint`

Сейчас в репозитории есть только стандартные заглушки `ExampleTest` (в `tests/Feature` и `tests/Unit`, генерируются фреймворком по умолчанию) — реального тестового покрытия функциональности магазина/админки нет.

## Архитектура

Стандартный Laravel MVC, вся логика находится прямо в контроллерах (нет слоя сервисов/репозиториев, нет Form Request — валидация, если она есть, выполняется прямо в контроллерах).

**Доменная модель** (`app/Models`):
- `Product` принадлежит `Type` (категории).
- `Basket` — это корзина; `ProductsInBasket` — модель строки корзины по типу pivot (`basket_id`, `product_id`, `count`), связывающая `Basket` и `Product`.
- `Order` принадлежит `Basket` и `Status`; заказ хранит снимок данных доставки/оплаты (`name`, `phone`, `email`, `date`, `time`, `dostavka`, `type_oplata`), а не ссылку на `User`.
- `Comment` — отзыв покупателя со своим `Status`.
- В `BasketController` не видно привязки корзины к пользователю/сессии — используется `Basket::where('active', 1)->first()`, то есть одна единая "активная" корзина на всё приложение. Учитывай это при изменении логики корзины/заказа: изоляции корзин между пользователями нет.

**Маршруты** (`routes/web.php`): все маршруты витрины и админки — плоские неавторизованные записи `Route::get/post` (без групп маршрутов, без auth-middleware), напрямую ведущие к методам контроллеров — например, `/` → `ProductController::getAll`, `/corsina` (корзина) → `BasketController`, `/admin` → `AdminController`, `/orders/export` → `OrderExportController`. Имена маршрутов активно используются в Blade-шаблонах и редиректах (`route('corsina')`, `route('admin')` и т.д.) — перед переименованием маршрута сверяйся с `web.php`, чтобы не сломать соответствие имя↔URI.

**Контроллеры** (`app/Http/Controllers`):
- `ProductController` — список товаров витрины/фильтрация по типу, страница карточки товара.
- `BasketController` — добавление/удаление/просмотр товаров в единственной активной `Basket`.
- `OrderController` — создаёт `Order` из корзины/формы оформления заказа.
- `CommentController` — приём и отображение отзывов покупателей.
- `AdminController` — админ-панель: список заказов с сортировкой/фильтрацией по дате и статусу, смена статусов (при этом отправляются письма покупателю через `Mail::raw` — см. `setStatus`), модерация отзывов.
- `OrderExportController` — экспорт заказов (вместе со строками корзины) в `.xlsx` через `phpoffice/phpspreadsheet`, отдаётся как скачивание временного файла.

**Представления** (`resources/views`, Blade, `.blade.php`): `index` (витрина), `card` (карточка товара), `corsina` (корзина), `dostavka` (информация о доставке), `comments`/`feedback` (отзывы), `admin` (админ-панель). Именованные маршруты активно используются внутри этих шаблонов — перед переименованием маршрута ищи его использование через grep по view.

**Фронтенд-ассеты**: обычная настройка Vite (`vite.config.js`, `resources/css`, `resources/js`) с `laravel-vite-plugin` — без Vue/React, без TypeScript.

Примечание: по всему проекту встречаются русские идентификаторы (имена маршрутов/параметров вроде `corsina` = корзина, `dostavka` = доставка, `otziv`/`comment` = отзыв, `type_oplata` = тип оплаты) — сохраняй эти названия как есть ради консистентности, не переводи их.
